# AI 协作开发指南

> 借助 AI（Cursor、Claude 等）高效学习 Java、Nginx 等技术。

## 核心理念

**文档 → 代码 → 学习** 的循环：

```
1. 让 AI 生成设计文档（理解整体）
2. 让 AI 实现代码（看具体实现）
3. 读代码 + 提问（深入学习）
4. 写学习笔记（内化知识）
```

---

## 协作流程

### 1️⃣ 需求描述

向 AI 描述时包含三要素：

| 要素 | 说明 | 示例 |
|------|------|------|
| **背景** | 解决什么问题 | 「实现文章列表，从 Java 后端拉数据」 |
| **范围** | 做什么/不做什么 | 「只做列表页，详情页后续再实现」 |
| **约束** | 技术栈/规范 | 「Next.js App Router，服务端组件优先」 |

### 2️⃣ 文档先行

新功能先让 AI 生成设计文档：

```bash
# 提示词示例
"我要实现文章列表页，请先写设计文档：
- API 接口设计（请求/响应）
- 数据流程（前端 → Nginx → Java）
- 组件结构
文档放在 docs/learning/article-list.md"
```

### 3️⃣ 代码实现

基于文档让 AI 生成代码：

```bash
# 提示词示例
"根据设计文档实现代码，要求：
- 关键处加学习注释（如 '// 依赖注入：Spring 自动管理 Bean'）
- 命名自解释，避免冗余注释
- 完成后说明如何测试"
```

### 4️⃣ 学习内化

代码完成后，通过提问深入学习：

```bash
# 学习 Java
"这段 Controller 代码如何体现 Spring 的依赖注入？"
"@Service 和 @Component 有什么区别？"

# 学习 Nginx
"为什么这里用反向代理而不是直接调用？"
"proxy_pass 和 rewrite 的作用是什么？"

# 学习架构
"数据从前端到后端的完整流程是什么？"
```

## 提示词模板

### 初始化项目

```
我想学习 [技术]，请帮我：
1. 搭建 [技术栈] 脚手架
2. 创建最简可运行示例
3. 在关键处加学习注释
4. 生成 docs/learning/[技术]-basics.md 学习笔记
```

### 添加功能

```
我要实现 [功能]，请：
1. 先写设计文档（接口/流程/数据结构）
2. 基于文档实现代码
3. 关键逻辑加学习注释
4. 说明如何测试
```

### 修复 Bug

```
遇到错误：[错误信息]
相关代码：[文件路径或代码片段]
请：
1. 分析根因
2. 给出修复方案
3. 说明为什么会出现这个问题
```

### 学习代码

```
请帮我理解这段代码：
1. 核心逻辑是什么？
2. 用了哪些技术/设计模式？
3. 如果我要加 [功能]，应该改哪里？
```

## 最佳实践

✅ **推荐**
- 小步迭代，每步验证后再继续
- 代码完成后写学习笔记
- 用提问驱动学习（「为什么这样设计？」）
- 重要决策记录在文档中

❌ **避免**
- 一次性生成大量代码不阅读
- 只复制代码不理解原理
- 文档和代码不一致
- 过度设计（这是学习项目）

## 常见问题

**Q：AI 生成的代码看不懂？**  
A：让 AI 逐行解释，或问「这段代码的核心逻辑是什么？」

**Q：如何确保代码质量？**  
A：运行 `pnpm build` 和 `pnpm lint`，并要求 AI 解释警告信息。

**Q：学完一个模块后如何复盘？**  
A：让 AI 帮你生成该模块的架构图 + 数据流图，写在笔记中。
